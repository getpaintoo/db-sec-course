# Практика 3. Краткий отчет

## Что настроено

- создан лабораторный `pg_hba.conf` с правилами для `trust`, `md5` и `scram-sha-256`;
- созданы пользователи `user_scram`, `user_md5`, `user_alice`;
- включено логирование подключений и отключений;
- подготовлен production-вариант `pg_hba.production.conf`.

## Что проверяется

1. Пути к `hba_file`, `config_file`, `data_directory`.
2. Типы хешей паролей в `pg_shadow`.
3. Поведение правил `trust`, `md5`, `scram-sha-256`.
4. Готовность к миграции пользователей с md5 на SCRAM.

## Вывод

Для production рекомендуется `scram-sha-256` плюс ограничение по IP и обязательный аудит логов. Метод `trust` допустим только как короткий лабораторный сценарий.
