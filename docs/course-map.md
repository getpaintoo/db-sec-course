# Карта курса

Структура репозитория приведена к тем практикам, которые опубликованы на сайте курса:

1. `Практика 1: Анализ и классификация потенциальных угроз`
2. `Практика 2: Сравнительный анализ моделей доступа и проектирование RBAC-модели`
3. `Практика 3: Настройка различных методов аутентификации`
4. `Практика 4: Создание иерархии ролей (администратор, менеджер, пользователь)`
5. `Практика 5: Детальная настройка привилегий для ранее созданных ролей`
6. `Практика 6: Реализация политики RLS, позволяющей пользователям видеть только свои данные`

Принятые проектные соглашения:

- практика 2 использует БД `task_management`;
- практики 4-6 используют БД `corporate_tasks`;
- для практики 3 контейнер PostgreSQL стартует с лабораторным `pg_hba.conf`, а production-вариант вынесен отдельно в `scripts/03_authentication/pg_hba.production.conf`.
