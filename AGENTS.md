# Инструкции для Codex / AI-агента

Цель репозитория: хранить решения по курсу «Безопасность баз данных» в удобном виде для Git.

## Правила работы

1. Не менять общую структуру папок без необходимости.
2. Для каждой практики держать:
   - `README.md` с кратким объяснением;
   - `setup.sql` или основной SQL-файл;
   - `verify.sql`, если нужно показать проверку результата.
3. Не смешивать решения разных практик в одном SQL-файле.
4. При изменении прав, ролей или RLS всегда добавлять проверочный сценарий.
5. Если требуется дополнить отчет — редактировать markdown-файлы в `theory/`.
6. Коммиты делать понятными и короткими.

## Быстрые команды

```bash
docker compose up -d
docker compose exec -T -u postgres db psql -U postgres -d postgres -f /workspace/scripts/04_role_hierarchy/setup.sql
```

## Ожидаемый стиль

- аккуратные SQL-скрипты;
- минимум магии, максимум явных `GRANT`, `REVOKE`, `CREATE POLICY`;
- комментарии только там, где они реально помогают понять логику.
